# Règles de travail

- Ne pas modifier un fichier existant sans expliquer le changement.
- Proposer le code avant modification.
- Ne pas supprimer de code sans validation explicite.
- Travailler par petites étapes.
- Après chaque modification, demander confirmation avant de continuer.
- Respecter strictement la structure du projet.
- Ne jamais écraser un fichier complet sans sauvegarde préalable.
- Ajouter des commentaires clairs dans le code.
- Indiquer toujours quels fichiers sont modifiés.
- Optimiser pour compatibilité Netlify (site statique).
- Après chaque étape validée : proposer un commit Git avec un message clair.
